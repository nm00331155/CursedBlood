【1. 画面最大化 / カメラ改善】
================================

現在の問題:
- ゲーム画面が狭い
- UIが場所を取りすぎる
- プレイヤーや地形が小さく見づらい

対応:
- プレイ領域をスマホ画面全体に近いサイズまで拡大する
- UIは別枠ではなくオーバーレイ化する
- Camera2D をより強くズームインする
- 体感として最低でも今より約2倍近い拡大感を目指す
- 地形ブロックが画面をしっかり埋める印象にする
- プレイヤーが小さすぎて見えない問題を改善する
- Camera2D の zoom / smoothing / lookAhead は inspector で調整可能にする
- プレイヤー近傍の視認性を少し上げてもよい

実装候補:
- GameCamera.cs
- MainGame.tscn
- CanvasLayer / HUD レイアウト

【3. 回収ポイント仕様変更】
================================

現状の「触れた瞬間ゲーム終了」は廃止する。

新仕様:
- 回収ポイントに近づくと帰還ボタンを表示
- 帰還ボタン押下で確認ダイアログを表示
  - 「帰還する」
  - 「まだ続ける」
- 回収ポイントは「撤収可能状態を与える地点」とする
- 接触時に即終了しない
- 小通知やSEを出してよい
- ミニマップ上でも目立つようにする

自動移動との相性対策:
- 回収ポイント近接中は移動をやや減速
- ニュートラル入力で止まりやすくする
- 帰還ボタン押下中は移動入力を一時停止してよい
- ダイアログ表示中は必ず移動停止

================================
【4. 下降チェインシステム】
================================

新システムとして「下降チェイン」を実装する。

概要:
- 地中に通過チェックポイントを生成
- 次のチェックポイントは基本的に現在地より下方向に生成
- 左下/右下に少し振ってよい
- チェックポイントを通過するとチェイン開始または継続
- 次のチェックポイントを 5秒以内 に通過すると継続
- 通過できなければチェイン終了
- 最大チェイン数は無限

生成ルール:
- 現在地点より必ず下
- 真下固定でなくてよい
- 最短ルートならギリギリ間に合う距離感にする
- 目安として最適移動で 3.5〜4.5秒程度
- 到達不能や理不尽配置は避ける
- ミニマップや方向表示で追えるようにする

見た目:
- 地中で目立つ発光リングや通過マーカー
- 残り猶予が少ない時は点滅
- ミニマップ上にも表示
- 通過時に小さなSEやポップアップ

================================
【5. チェイン報酬】
================================

重要:
通常の経済倍率上昇はかなり低く抑えること。
チェインは「主目的の一つ」ではあるが、経済破壊を起こさないことが最優先。

通常報酬:
- 1チェインごとに換金倍率 +0.15%（仮設定）
- 後で調整できるようにする

節目ボーナス:
- 50チェインごとにやや大きめの節目ボーナス
- 100, 150, 200... も継続
- 内容は設定で調整可能にする

================================
【6. チェインバフ】
================================

チェインの主報酬はバフ寄りにする。

バフ仕様:
- バフは重複可能
- 最大5スタック
- 各スタックは個別タイマー管理
- 重複時間は少しだけ重なる程度
- 後半ほど真価を発揮する設計
- スタック数・持続時間・効果量は inspector/config で調整可能

仮ルール:
- 5チェインごとに +1秒（残り時間 or 酸素猶予）
- 10チェインごとに移動速度バフ1スタック
- 10チェインごとに掘削速度バフ1スタック
- 20チェインごとにソナーバフ1スタック
- 50チェインごとに節目ボーナス

推奨:
- 移動速度バフと掘削速度バフは別管理
- 1スタックあたり +4〜6% 程度の控えめ設定
- 最大5スタック
- 1スタックごとに 4〜6秒程度
- スタックごとに個別タイマーを持つ

================================
【7. ソナー再設計】
================================

下降チェイン追加後もソナーが死なないようにする。

ソナーの役割:
- 次チェイン目標の方向精度を上げる
- 宝の方向を示す
- 回収ポイントの方向を示す
- 危険（敵密度など）の方向/強さを示す
- 帰るか、進むか、寄り道するかの判断材料にする

方針:
- チェインだけ追えば最適、にはしない
- ソナーがあるとチェイン継続率が上がる
- ソナーがあると高価値選択や安全判断がしやすい

実装案:
- 次チェイン目標はソナーがあるほど方向表示が正確
- 敵が多い方向は危険反応を出す
- 回収ポイント方向も探知可能
- 必要ならソナーバフで範囲/精度/更新頻度を上げる

================================
【8. 敵実装】
================================

敵は完全戦闘型ではなく、基本は避けるべき存在とする。
ただしドリルで倒すことは可能にする。
しかし、倒すと必ず何らかの被害を受ける仕様にする。

重要ルール:
- 敵を安全に倒せる仕様にはしない
- 「倒す = 必ず代償がある」
- よって、無理に倒す選択肢を減らす
- 敵撃破を稼ぎの主軸にはしない
- 敵は進行妨害・判断圧・チェイン中のリスク要素

基本仕様:
- ドリル前方接触で敵を倒せる
- ただし倒した瞬間または接触時に、必ず何らかの被害をプレイヤーへ与える
- 被害内容はゾーンや敵タイプで分けてよい

被害候補:
- 通常ダメージ
- 毒状態
- 移動速度低下
- 上方向への押し戻し
- 酸素減少
- 一時的な掘削速度低下

初期実装:
- まずは1〜2種類の敵でよい
- 深度が深いほど出現率を上げる
- 高チェイン中は追加で敵圧を上げる
- 処理負荷や実装負荷を抑えつつ、リスクとして機能させる

================================
【9. 深度 / チェインによるリスク増加】
================================

下に行くことが得ばかりにならないよう、明確なリスクを持たせる。

実装方針:
- 深く行くほど敵が増える
- 高チェイン時ほど敵圧が増える
- 深く行くほど回収ポイントの生成率または近接しやすさを下げる
- 欲張るほど帰還難易度が上がる

推奨:
- 危険度 = 深度ベース + 現在チェイン数ベース
- 回収ポイントは完全に消すのではなく、深部ほど遠く・少なく感じる程度に抑える

================================
【10. 失敗コスト】
================================

失敗時の回収費用/救助費用を導入または強化する。

方針:
- 深く潜るほど失敗コスト増加
- 所持資源が多いほど失敗コスト増加
- 高チェインほど失敗コスト増加

イメージ:
失敗コスト = 基本救助費 + 深度補正 + 所持品補正 + チェイン補正

目的:
- 欲張って深く潜るほど、事故時に利益が薄くなる/赤字化しうる
- 「今帰るべきか」の判断を生む
