# CursedBlood Phase 1 修正指示（最優先・全面上書き前提）

あなたは Godot 4.6.1 (.NET/C#) プロジェクト `CursedBlood` の実装担当です。  
この修正では **docs/01_PHASE1_GRID_MOVEMENT.md を唯一の正とし、現状コードの旧仕様・別フェーズ混在部分を整理して、Phase 1最小プレイアブルを完成** させてください。

## 最重要方針

- **必ず `docs/01_PHASE1_GRID_MOVEMENT.md` と `docs/00_PROJECT_SPEC.md` を先に読んでから実装すること**
- 旧仕様（140pxブロック、7列、4方向、GridManager中心設計）は **採用しない**
- 現在リポジトリにある Phase 2〜9 相当の依存が Phase 1 の妨げになっている場合、**Phase 1完了を優先して切り離す/無効化する**
- **旧コード（`Scripts/Sample/`, `Scripts/Core/`）は一切変更しない**
- 新規コードは **`Scripts/CursedBlood/*` と `Scenes/CursedBlood/*` のみ**
- namespace は **`CursedBlood.*`**
- パフォーマンス最優先：
  - **byte配列**
  - **16行/チャンク**
  - **Empty非描画**

  - **不要なGCを避ける**
- 実装後は **dotnet build**, **Godot起動確認**, 必要なら **APK再ビルド** まで行うこと
- 最終報告は日本語で詳細に行うこと

## 実装仕様（厳守）

# 1. CellType.cs
byte enum で実装すること。

想定:
- Empty = 0
- Dirt = 1
- Stone = 2
- HardRock = 3
- Ore = 4
- Bedrock = 5
- Enemy = 6
- Boss = 7

また `CellTypeUtil` のような静的ユーティリティを用意し、最低限以下を持つこと:
- `GetHardness(CellType type)`
- `IsDiggable(CellType type)` ※ Bedrock以外true
- `GetColor(CellType type, int depthTier)`

---

# 2. ChunkData.cs
- Width = 67
- Height = 16
- `byte[] Cells`
- `int ChunkIndex`
- `int StartRow => ChunkIndex * Height`
- `GetCell(localCol, localRow)`
- `SetCell(localCol, localRow, value)`
- `ToIndex(col, row)`

**1セル = 1バイト**
CellDataクラスのようなオブジェクト粒度管理は使わないこと。

---

# 3. TerrainGenerator.cs
深度に応じて Dirt / Stone / HardRock / Bedrock / Empty を生成すること。  
docs の深度別分布を反映すること。

最低限ほしいこと:
- 左右端の一部に Bedrock 壁
- ときどき横断 Bedrock 帯
- ただし **必ず掘削可能な開口部を残す**
- 各行に最低限の通行可能エリアを保証
- 開始エリア（最初の2チャンク）は Empty 中心

**Simplex等の重い生成ではなく軽量な擬似ノイズでよい**

---

# 4. DigHelper.cs
- `DigShape` enum を用意（Square / Diamond / Fan）
- Phase 1 では `width=5`, `shape=Square` 固定
- `GetDigArea(playerPos, direction, width, shape, playerSize)` を実装
- 8方向それぞれで、進行方向の前面に対して幅5の掘削範囲を返すこと
- `ExecuteDig(ChunkManager chunks, ...)` で Bedrock以外を Empty 化

**重要**:
- 「1セルだけ壊す」のではなく、**進行方向に対して幅5で面掘り**
- 将来 shape 拡張しやすい形にする

---

# 5. ChunkManager.cs
Node2D 継承。  
`GridManager` ではなく **`ChunkManager` を主役にすること。**

定数:
- `CellSize = 16`
- `ChunkHeight = 16`
- `Columns = 67`
- `VisibleRows = 87`
- `FieldOffsetX = 4f`
- `FieldOffsetY = 200f`

内部:
- `Dictionary<int, ChunkData> _chunks`
- `TerrainGenerator _generator`

機能:
- `Initialize()`
- `GetChunk(int chunkIndex)`
- `GetCell(int col, int absoluteRow)`
- `SetCell(int col, int absoluteRow, byte value)`
- `UpdateCamera(int playerRow)` or 同等責務
- `GridToScreen(...)` / `GridToWorld(...)`
- `Reset()`

描画最適化:
- 表示範囲の行だけ描画
- **Empty は描画しない**
- 同一 CellType の水平連続セルを 1 DrawRect にまとめる
- 背景は1回だけ塗る

**チャンク破棄も実装**  
上方向に大きく離れた不要チャンクは破棄すること。

---

# 6. PlayerStats.cs
純粋データクラス寄りにする。

最低限:
- `MaxLifespan = 60f`
- `CurrentAge`
- `MaxHp = 100`
- `CurrentHp`
- `GridPosition = (33, 8)` 初期位置
- `PlayerSize`
- `MaxDepthPixels` または `MaxDepthRow`
- `BaseMoveInterval = 0.02f`
- `DigWidth = 5`
- `DigShape = Square`
- `Generation = 1`

算出:
- `HumanAge`
- `LifePhase`（Youth / Prime / Twilight）
- `PhaseMultiplier`（0.6 / 1.0 / 0.7）
- `EffectiveMoveInterval`
- `IsAlive`

フェーズ遷移:
- Youth: 3x3
- Prime: 5x5
- Twilight: 5x5
- Youth→Prime 時にサイズ変更

メソッド:
- `AdvanceTime(float delta)`
- `TakeDamage(int damage)`
- `CalculateScore()`
- `Reset()`

---

# 7. PlayerController.cs
Node2D 継承。

**最重要修正ポイント**
現状の4方向実装をやめて **8方向入力** にすること。

### 入力
- 矢印キー同時押しで斜め
- スワイプ/ドラッグでも8方向
- 自由角度から最も近い8方向へスナップ
- 一度方向指定したら次入力までその方向へ自動継続
- 先行入力バッファあり
- Space 長押しで Guard（移動停止）

### 移動
- 0.02秒/セル を基準
- Stone通過時 1/2
- HardRock通過時 1/4
- Bedrock 方向は進めず停止
- **プレイヤー占有サイズ（3x3 / 5x5）を考慮**
- **プレイヤー本体が壁にめり込まない**
- 移動先の占有範囲を検証
- 移動直前/移動中に `DigHelper` を使って前面を掘削

### 描画
- 円ではなく、最低限 **PlayerSize × CellSize の矩形ベース** でよい
- Youth / Prime / Twilight で色変更
- 方向インジケータ
- Guard中表示

**注意**:
- 今の `DrawCircle` ベースや1セル前進前提の処理は docs と違うので見直すこと
