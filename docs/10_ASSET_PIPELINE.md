# アセット管理（仮→本番の差し替え手順）

## 目的

CursedBlood の開発では、Phase進行中は仮アセットで高速実装し、後から本番用アセットへ差し替える。
今回の新仕様では、旧設定の呪い・老化・世代関連アセットは不要となり、その分
ソナー・酸素・回収ポイント・借金関連UIを優先する。

## 方針

- 初期実装では `_Draw()` や単色矩形で仮表示してよい
- 本番前に必要なアセットを差し替える
- アセット差し替えでロジック依存を増やさない
- ファイル名・命名規則を固定して差し替えしやすくする

## 優先度の高いアセット

### 最優先
- プレイヤー仮スプライト
- ブロック見分け色
- 酸素UI
- ソナーUI
- 回収ポイントアイコン / オブジェクト
- リザルト画面UI
- 借金 / 返済UI

### 次点
- 敵仮スプライト
- ボス仮スプライト
- 掘削エフェクト
- 被弾エフェクト
- 回収成功 / 救助演出

## 削減できるアセット

旧仕様で不要になったもの:
- 呪い演出
- 老化差分
- 世代継承演出
- 家系図専用ビジュアル
- 享年演出
- 呪い研究専用演出

## スプライト方針

### プレイヤー
- 最初は1シートでよい
- 9フレーム構成可
- 方向差分は後回し
- 方向感はエフェクト・UIで補完してよい

### 敵
- 仮色矩形 → 仮スプライト → 本番差し替え

### 回収ポイント
- 目立つが世界観に合う見た目
- ソナーと相性がよいこと

## 差し替えルール

- 仮アセットと本番アセットでアンカーやサイズを極力一致させる
- スプライトシート差し替え時にコード修正が最小になるよう設計する
- UIテクスチャ差し替え時もノード構造は維持する

## 注意事項

- 見た目より仕様検証優先
- 早い段階で操作性・視認性・状態悪化演出だけは最低限確保する
