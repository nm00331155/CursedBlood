Godot 4.6.1 (.NET/C#) の CursedBlood プロジェクトについて、現在のプレイ感を大きく見直してください。
目的は「スマホ縦画面で、もっと広く掘れて、もっと分かりやすく、もっと速く気持ちよく遊べる掘削ゲーム」にすることです。

# 最重要方針
現状は以下の問題がある。
- 地形の意味が分かりにくい
- 掘る範囲が狭く感じる
- 進めない場所が多くテンポが悪い
- 画面が遠すぎて掘削ゲームの爽快感が弱い
- 左右の壁で世界が閉じて見え、宝探し感が弱い
- ミニマップがなく、どこを掘って進んだか把握しづらい
- 上方向の地上判定がおかしく、地上からさらに上へ行けてしまう
- スピード感が足りない

今回の改修では、以下のユーザー要望を強く反映すること。

# 追加要望（優先度高）
1. 敵はまだ実装されていない前提で整理する
2. 左右の壁は不要。無限に広がる地中を好きに掘れるようにする
3. 通れない道は不要。テンポを悪くするだけなので、極端な行き止まりや理不尽な進行不能を減らす
4. 画面全体に常に掘る場所が広がっている印象にする
5. ゲームの拡大率をもっと上げる。最低でも今の2倍近い体感で、ブロックが画面をしっかり埋めるくらいまで寄せる
6. UI右上にミニマップを追加し、スタート地点から掘り進んだ道筋を表示する
7. 地上で上を押すと空まで行けてしまう問題を修正する
8. もっとスピード感のあるゲームにする

# 敵について
- 現時点では敵が未実装なら、敵前提の進行阻害・警戒UI・説明は外すか最小化する
- ソナーは敵ではなく、回収ポイント・鉱脈・重要物の検知中心でよい
- HUD / テキスト / デバッグ表示に敵前提の文言があれば整理する

# 1. ワールド構造を「左右に広がる」方向へ変更
## 目的
狭い縦坑ではなく、無限に広がる地下を自由に掘る宝探し感を強める。

## 対応
- 左右の固定壁をやめる
- ワールドを左右に継続生成できる構造に変更する
- 必要なら ChunkManager / TerrainGenerator を横方向にもチャンク化する
- 少なくともプレイヤー周辺の一定範囲を動的生成し、左右に掘り進められるようにする
- スタート地点付近だけは分かりやすい初期空間を用意し、その先は横にも下にも広がる
- 画面端で不自然な「ここで終わり」感を出さない
- Bedrock 的な絶対壁で左右を塞がない

## 注意
- 完全無限が重すぎるなら、実質無限に感じる範囲を動的ロード / アンロードで実現する
- メモリ負荷に注意
- 横方向へ広がってもミニマップで自分の位置が分かるようにする

# 2. 「通れない道」を減らしてテンポ改善
## 目的
詰まりや理不尽感でテンポが落ちるのを防ぐ。

## 対応
- 極端に掘れない地形の密度を下げる
- Bedrock / 進行不能セルは最小限にする
- HardRock がある場合も、全面的に行き止まりを作りすぎない
- 地形生成で「常にどこかしら掘り進める余地がある」ようにする
- プレイヤーの掘削幅と占有サイズを前提に、進路候補が消えすぎないよう補正する
- 必要なら初期段階は「全て掘れる / 一部硬いだけ」くらいまで緩和してよい
- 行き止まりを作るより、分岐と広がりを増やす

## 実装の考え方
- 一本道の閉塞感ではなく、周囲に掘れる候補が広がっていることを重視
- テンポ優先で、ゲームとしての気持ちよさを優先
- 今後難易度を上げるとしても、現時点ではストレスより爽快感を優先

# 3. 地形を「画面全体に掘る場所が広がる」見た目にする
## 目的
今の画面は細かくて閉塞感が強く、「掘れる世界が広がっている」感じが弱い。

## 対応
- 地形パターンを見直し、周囲一帯が掘削対象で埋まっている印象にする
- 黒い帯や強い仕切りが多すぎるなら減らす
- 空洞の作り方を見直し、スタート周辺以外は「掘って広げていく」感を出す
- 価値資源や回収ポイントを点在させ、あちこち掘りたくなる構成にする
- 通路を既成するより、「地層そのものを掘り崩して進む」方向へ寄せる
- 視認性改善は維持しつつ、全体としてフィールドに余白感が出すぎないようにする

# 4. カメラ / 表示倍率を大幅に上げる
## 目的
今の表示は遠すぎる。もっとブロックが画面を埋めるサイズ感にする。

## 対応
- カメラを今より大幅にズームインする
- 体感として最低でも現状の2倍近く寄った印象になるよう調整する
- プレイヤー周辺の掘削判断を優先し、遠景情報は削ってよい
- ブロックが画面をしっかり埋める密度にする
- プレイヤーが地形の中で埋もれず、掘削対象がはっきり見えるようにする
- 可能ならズーム値は設定しやすいよう外出しする

## 受け入れ条件
- 以前よりブロック1つ1つが明らかに大きく見える
- キャラが視認できる
- 掘る対象が分かる
- 「フィールド全体の縮小俯瞰」ではなく「掘る現場にいる」感が出る

# 5. ミニマップ追加
## 目的
どこを掘って進んだか分からず、広がる地下の探索感が弱い。

## 対応
- UI右上にミニマップを追加する
- スタート地点を基準点として表示する
- プレイヤーが掘って通った道筋を表示する
- 現在位置が分かるようにする
- 可能なら回収ポイント、重要物、地上位置も簡易表示する
- 初期実装では超簡易でよい
  - 掘った場所: 明色
  - 未到達: 暗色
  - 現在位置: 強調色
  - スタート地点: 別色
- 更新は軽量に行う
- 全世界を毎フレーム描かず、探索済み範囲のみ管理してもよい

## UI条件
- 右上に配置
- HUDを圧迫しすぎない
- 小さくても道筋が読める
- 必要なら半透明パネル上に表示

# 6. 地上より上に行ける問題の修正
## 目的
地上で上を押すと空まで行けてしまうのは不自然でバグに見える。

## 対応
- 地上レベルより上へは進めないようにする
- 上方向入力時、地上境界で停止する
- ただし地上への帰還は可能にする
- 「帰還」に使うルールが別にあるなら、それと整合させる
- もし地上セル表現が必要なら、地上ライン / 出口ラインを分かるようにする
- 地上に戻ったら適切な帰還処理 / 回収判定 / 結果遷移へ接続しやすい構造にする

# 7. スピード感の強化
## 目的
もっとサクサク掘って進める感覚を出す。

## 対応
- 移動テンポを上げる
- 必要なら move interval を短縮する
- 掘削テンポを上げる
- 硬い地形の減速を弱める、または序盤はかなり軽くする
- ヒット時の反応を明確にする
- 掘削エフェクトや画面揺れはやりすぎない範囲で追加してもよい
- 入力から移動開始までの遅れを減らす
- 方向転換のもたつきを減らす
- 進める方向が分かるよう、視認性改善・掘削可否ハイライト・進路ヒントも合わせて改善する

## 目指す体感
- 迷って止まる時間を減らす
- 掘れば進む
- 行き先が見つかる
- 深く潜るほど広く掘り散らかしたくなる
- 60秒の中で「もたつく」より「駆け抜ける」感覚を優先

# 8. 既存の視認性改善も継続して実装
以下は前回方針を継続して反映すること。
- 掘れる / 硬い / 掘れない の色分離
- 掘削可否プレビュー
- プレイヤー周囲の進路候補表示
- プレイヤー視認性強化
- HUD文字サイズ拡大
- 不要な常時説明削減
- 動的バーチャルパッド改善
- ソナー可視化

ただし今回の新要望を優先し、
「広く掘れること」「テンポが良いこと」「画面が近いこと」「ミニマップで経路が分かること」を特に重視すること。

# 9. UI再構成
## 目的
画面が狭いのに情報が細かく、読みづらい。

## 対応
- 右上にミニマップを置く前提で HUD を再整理する
- テキストは全体的に拡大する
- 常時表示の操作説明文は減らす
- プレイ中は必要な情報だけ表示する
- 文字サイズ・余白・パネルサイズをスマホで読めるようにする
- プレイエリアを最大限広く取る
- ただしミニマップと主要HUDは両立すること

# 10. 実装上の注意
- 既存構造を完全に壊すより、段階的に差し替えやすい形にする
- 動的生成・ミニマップ・視認性改善は調整可能パラメータを持たせる
- パフォーマンスに注意
- null参照を避ける
- 既存の docs / 新コンセプト（借金返済・短時間潜行・回収ポイント・ソナー）に整合させる
- 敵未実装なら、敵依存の進行阻害は作らない
